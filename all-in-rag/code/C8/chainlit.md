# 尝尝咸淡 RAG 系统优化方案

为了进一步提升系统的智能化水平、用户体验及工程性能，计划在下一阶段按照从底层架构到前端体验的顺序，对系统进行以下五个核心维度的迭代优化：

## 1. 知识库持久化与增量更新（已完成）
* **当前问题**：系统每次启动时会全量重新读取本地文档并进行切分。即使向量索引已经存在，数据准备环节依然耗时。随着食谱数量增加，系统启动速度将明显变慢。
* **优化目标**：
    1. 状态解耦：完善本地持久化机制，启动时直接挂载已存在的向量数据库，跳过全量文件读取和切分过程。
    2. 增量构建：引入文档哈希值或修改时间戳等文件比对机制。当触发更新时，系统自动比对并仅对新增或修改的食谱文件进行向量化和入库操作，从而降低算力与系统开销。

## 2. 引入历史对话记忆缓存
* **当前问题**：系统目前采用无状态的单次请求模式，无法读取用户之前的对话记录，导致交互缺乏连贯性。
* **优化目标**：设计并接入系统记忆缓存机制，如会话缓存或短时记忆窗口。在构建提示词时动态携带最近多轮的历史对话记录，让系统具备存储历史对话上下文的数据基础。

## 3. 提升多轮对话上下文关联能力
* **当前问题**：受限于无状态设计，系统缺乏上下文指代消解能力。例如用户先询问红烧肉是什么菜系，系统回答后用户紧接着询问具体做法，此时系统会丢失主语，从而给出错误的菜谱回答。
* **优化目标**：基于历史对话记忆缓存，进一步引入上下文改写技术。使模型在处理当前问题时能够自动结合上一轮对话中的实体信息，精准理解用户的连贯意图并给出正确回答。

## 4. 思考链展示与加载体验优化
* **当前问题**：在模型检索文档和生成长文本的计算期间，页面呈现空白状态，容易导致用户误判系统停止响应。
* **优化目标**：
    1. 界面展示：增加动态加载动画或骨架屏设计。
    2. 逻辑反馈：展示模型的推理链条，例如在界面输出正在检索菜谱库或正在分析制作步骤等系统状态，通过展示处理过程填补用户的等待时间。

## 5. 拓展多模态视觉理解能力
* **当前问题**：系统目前仅支持纯文本交互，数据输入维度较为单一。
* **优化目标**：集成多模态大模型的视觉解析能力。支持用户上传图片作为输入，例如上传拍摄的菜品照片或现有的食材组合图片，系统通过图像识别内容并反向检索对应的菜谱和制作方法。

---
**阶段总结**
通过优先解决底层工程的持久化与增量更新问题，进而完善上下文记忆与逻辑关联，最后优化交互等待体验并引入多模态视觉能力，系统将从基础的文档检索工具升级为具备高可用性和多模态交互能力的智能辅助系统。
