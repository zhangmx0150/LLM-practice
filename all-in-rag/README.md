## 一、 什么是 RAG

### 1.1 核心定义

**RAG (Retrieval Augmented Generation)** 将模型已经学到的知识和外部查阅的知识相结合。

在 LLM 生成文本前，先通过检索机制从外部知识库中动态获取相关信息，并将这些“参考资料”融入生成过程，从而提升输出的准确性和时效性。

> **总结：** RAG 就是让 LLM 学会**“开卷考试”**，它既能利用自己学到的知识，也能随时查阅外部资料。

### 1.2 技术原理

#### （1）检索阶段 (Retrieval)

1. **知识向量化**：嵌入模型 (Embedding Model) 将外部知识库编码为向量索引，存入向量数据库。

2. **语义召回**：当用户发起查询时，检索模型利用同样的嵌入模型将问题向量化，并通过相似度搜索，从海量数据中精准锁定与问题最相关的文档片段。

#### （2）生成阶段 (Generation)

1. **上下文整合**：生成模块接收检索阶段送来的相关文档片段以及用户的原始问题。

2. **指令引导生成**：该模块遵循预设的 Prompt 指令，将上下文与问题有效整合，并引导 LLM 进行可控的、有理有据的文本生成。

## 二、 为什么用 RAG

### 2.1 技术选型路径

按照对模型改动由小到大、成本由低到高的顺序，建议遵循以下路径：

**提示词工程 (Prompt Engineering) → 检索增强生成 (RAG) → 微调 (Fine-Tuning)**

* **提示词工程**：通过设计提示词引导模型，适用于任务简单、模型已有相关知识的场景。

* **RAG**：如果模型缺乏特定或实时知识，通过外部知识库为其提供上下文。

* **微调**：当目标是改变模型的“行为/风格/格式”（如何做）而非“知识”（知道什么）时，微调是最终选择。

### 2.2 关键优势

RAG 填补了通用模型与专业领域之间的鸿沟，有效解决 LLM 的局限性：

| 问题 | RAG 的解决方案 |
| :--- | :--- |
| **静态知识局限** | 实时检索外部知识库，支持动态更新 |
| **幻觉 (Hallucination)** | 基于检索内容生成，显著降低错误率 |
| **领域专业性不足** | 引入医疗、法律、政策等领域特定知识库 |
| **数据隐私风险** | 支持本地化部署，避免敏感数据泄漏 |

#### 🚀 RAG 四字优势口诀：准、新、省、活

* **准（准确可溯源）**：告别“胡说八道”。直接引用资料，每一句话都有据可查，提升严肃场景下的可信度。

* **新（知识零时差）**：支持“热插拔”。无需重训模型，新政策、新数据入库即用，保持时效性。

* **省（降本增效）**：省钱省算力。避免高昂的微调成本，利用外部知识辅助，小参数模型也能达到极佳效果。

* **活（灵活易拓展）**：万物皆可接入。支持 PDF、Word、网页等多种格式，且检索与生成解耦，方便模块化升级。

> **总结：** RAG 就是给大模型外接了一个**随时更新（新）**、**低成本（省）**、**支持多源文件（活）**的外部数据源，从而让它的回答**有理有据（准）**。

## 三、 如何使用 RAG

### 3.1 四步构建最小可行系统 (MVP)

1. **数据准备与清洗**：构建地基。标准化多源异构数据，采用语义段落切分策略，避免信息破碎。

2. **索引构建**：文本向量化并存入数据库，附加来源、页码等元数据，便于精确引用。

3. **检索策略优化**：采用**混合检索**（向量 + 关键词）提升召回率，并引入**重排序 (Rerank)** 模型精选结果。

4. **生成与提示工程**：设计 Prompt 模板，引导 LLM 基于上下文回答，并明确要求“不知道就说不知道”。

### 3.2 评估与优化

* **评估维度**：
    * **检索相关性**：找到的内容是否包含答案。
    * **生成质量**：语义准确性（意思对不对）与词汇匹配度（术语准不准）。

* **优化方向**：
    * **性能层面**：索引分层（高频数据缓存）、多模态扩展（图表检索）。
    * **架构层面**：从线性流程向复杂模式进化，如**并行分支模式**处理多路检索，或**循环模式**进行自我修正。
